# Beitragen zur AAS Suite

Vielen Dank für Ihr Interesse, zur AAS Suite beizutragen!

## 🎯 OSS-Schaufenster

Wichtig zu verstehen: Dieses Repository ist ein **One-Way-Mirror** (OSS-Schaufenster).

- Die Hauptentwicklung findet in einem privaten Repository statt
- Dieses Repository wird automatisch synchronisiert
- Pull Requests werden manuell in das private Repo übernommen

## 🔧 Pull Requests

### Ablauf

1. **Fork** - Forken Sie dieses Repository
2. **Branch** - Erstellen Sie einen Feature-Branch
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. **Commit** - Committen Sie Ihre Änderungen
   ```bash
   git commit -m 'Add amazing feature'
   ```
4. **Push** - Pushen Sie Ihren Branch
   ```bash
   git push origin feature/amazing-feature
   ```
5. **PR** - Erstellen Sie einen Pull Request

### Guidelines

- Beschreiben Sie Ihre Änderungen klar und ausführlich
- Fügen Sie Tests hinzu (falls zutreffend)
- Aktualisieren Sie die Dokumentation
- Folgen Sie dem bestehenden Code-Stil

## 🐛 Issues

Issues sind willkommen! Bitte:

- Beschreiben Sie das Problem klar
- Fügen Sie Schritte zur Reproduktion hinzu
- Geben Sie Ihre Umgebung an (OS, Version, etc.)

## 💡 Feature Requests

Feature-Vorschläge sind willkommen! Bitte:

- Beschreiben Sie den Use Case
- Erklären Sie, warum das Feature nützlich ist
- Geben Sie Beispiele (falls möglich)

## 📝 Code of Conduct

Wir erwarten von allen Mitwirkenden:

- Respektvolles und professionelles Verhalten
- Konstruktive Kommunikation
- Fokus auf die Sache

## ⏱️ Review-Prozess

Da dies ein Mirror-Repository ist:

- Reviews können etwas Zeit in Anspruch nehmen
- Änderungen müssen manuell ins private Repo übertragen werden
- Bitte haben Sie Geduld

## ❓ Fragen?

Bei Fragen:

- Öffnen Sie ein Issue
- Kontaktieren Sie das Team (siehe README.md)
